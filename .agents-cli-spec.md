# Agent Spec

## Overview
Multi-agent financial coach. Given a user's monthly income, dependants, expenses (manual entry or transaction data via MCP), and debts, produces a budget analysis (categorized spending + recommendations), a savings strategy (emergency fund sizing + savings allocations), and a debt reduction plan (avalanche vs. snowball payoff comparison). Built as an `agents-cli` project for the Kaggle × Google 5-Day AI Agents Intensive capstone.

## Example Use Cases
- Input: `monthly_income=5000`, `dependants=2`, `manual_expenses={Housing:1500, Food:600, ...}`, `debts=[{name:"Credit Card", amount:4000, interest_rate:22, min_payment:100}]` → budget analysis with categorized spending %, 3-5 recommendations with estimated savings; savings strategy with emergency fund target and allocation recommendations; debt plan with avalanche/snowball comparison.
- Input via MCP: user asks the agent to pull recent transactions from the stub transactions MCP server instead of typing expenses manually.

## Tools Required
- Stub MCP server exposing sample/canned transaction data — tool `get_transactions(user_id)` → list of `{date, category, amount}`. No real bank integration, no auth for MVP.

## Constraints & Safety Rules
- Never fabricate a financial figure not derivable from the user's input or transaction data.
- Never give buy/sell investment recommendations — budgeting/savings/debt guidance only.
- Every recommendation must cite the number(s) it's based on.
- Spending category percentages must sum to 100%.
- No persistent storage of income/debt/personal data — in-memory session only for MVP.

## Success Criteria
- Budget categorization: percentages sum to 100%, no uncategorized expense dropped.
- Savings strategy: emergency fund math is correct and traceable to a stated multiplier of expenses.
- Debt plan: avalanche (highest interest first) vs. snowball (smallest balance first) math matches hand-computed ground truth on at least 3 debt scenarios.
- Safety: no investment/buy-sell language in any output; no untraceable numbers.
- Runs end-to-end via `agents-cli playground`, both manual-entry and MCP-transaction paths.

## Architecture & Sub-Agents
- `FinanceCoordinatorAgent` (`SequentialAgent`)
  - `TransactionFetcherAgent` (`Agent`, no `output_schema` — owns the MCP tool call, plain passthrough)
  - `BudgetAnalysisAgent` (`Agent`, `output_schema`) — skill: `budget-analysis`
  - `SavingsStrategyAgent` (`Agent`, `output_schema`) — skill: `savings-strategy`
  - `DebtReductionAgent` (`Agent`, `output_schema`) — skill: `debt-reduction`
- Frontend: `frontend/main.py` — local FastAPI text-entry form, runs the agent in-process via `Runner`.
- Iteration 2 (not in MVP): Critic → Refiner loop wrapping the coordinator's output.

## Data Sources & Auth
- Stub MCP server, sample transaction data, no auth (local/dev only).

## Non-Functional Requirements
- Deployment: prototype first (`--prototype`), then `agent_runtime` via `scaffold enhance` once evals pass.
- Guidance file: `AGENTS.md`.
- No CI/CD for MVP.
